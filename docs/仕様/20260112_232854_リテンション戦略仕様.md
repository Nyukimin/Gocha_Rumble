# リテンション戦略仕様 - チャレンジ人口の拡大と継続人口の増加

**作成日**: 2026-01-12  
**バージョン**: 1.0  
**ステータス**: 初版

---

## 1. 概要

ゲームプロデューサーとして、**「チャレンジ人口の母数を増やす」**と**「継続人口をできるだけ増やす」**を最優先目標とした戦略的仕様を定義します。

**目的**: 長く遊び続けてもらうこと

---

## 2. チャレンジ人口の母数を増やす（新規ユーザー獲得戦略）

### 2.1 オンボーディングの最適化（最初の5分が全て）

**現状の問題**: 最初の体験が良くないと、すぐに離脱する

**仕様**:
- **即座にプレイ可能**: ログイン後、すぐに1回目のバトルを開始できる
  - チュートリアルは最小限（30秒以内）
  - 最初は「おすすめ編成」で自動スタート
  - 最初のバトルは必ず成功する（敵が弱い、または無敵時間あり）

- **最初の5分で「ハマる瞬間」を作る**
  - **1分目**: バトル開始、200体のユニットが動く様子を見る（視覚的インパクト）
  - **2分目**: ユニットが進化する瞬間を見る（「おお！」という驚き）
  - **3分目**: ダメージ数値が飛び散る様子を見る（ハクスラ感）
  - **4分目**: 最初のリザルト画面で報酬を獲得（達成感）
  - **5分目**: 獲得した報酬でユニットを強化（成長感）

- **「また遊びたい」と思わせる仕掛け**
  - 最初のバトル終了後、必ず「新しいユニットをアンロックしました！」と表示
  - 「次回はこのユニットを使えます」という期待感を演出
  - ログインボーナスを即座に付与（「明日も来るともっと良い報酬が！」）

### 2.2 ソーシャル要素による拡散（口コミ・共有）

**現状の問題**: 放置ゲームは拡散しにくい

**仕様**:
- **スクリーンショット・動画共有機能**
  - バトル中の「かわいい瞬間」をワンクリックでスクリーンショット
  - 進化の瞬間、大量のダメージ数値など「見せたい瞬間」を自動でハイライト
  - SNS共有ボタン（Twitter、Discordなど）を配置
  - 共有時に「Gocha Rumble 3Dで遊んでいます！」というテキストを自動付与

- **リプレイ動画の自動生成**
  - バトル終了後、自動で30秒のハイライト動画を生成
  - 特に印象的な瞬間（進化、大量撃破など）を自動で編集
  - 動画をダウンロード・共有可能に

- **「友達招待」システム**
  - 友達を招待すると、両方に報酬（レアユニット、コインなど）
  - 招待コードを簡単に共有できる（URL、QRコード）
  - 招待数に応じた特別報酬（「10人招待でレジェンドユニット」など）

- **ランキングの可視化**
  - 週間ランキングのトップ10を大きく表示
  - 「あなたの友達の中で○位」という表示
  - ランキング上位者のチーム編成を見られる（「この編成すごい！」という発見）

### 2.3 低ハードルでの参入（誰でも始められる）

**仕様**:
- **ブラウザで即プレイ可能**: インストール不要、アカウント作成も簡単（Google認証のみ）
- **軽量な動作**: 低スペックPCでも30FPS維持
- **無料で完結**: 課金要素は「時間短縮」「見た目変更」のみ（ゲーム性には影響しない）

### 2.4 マーケティングフックの設計

**仕様**:
- **「200体のメカが同時に戦う」という視覚的インパクト**
  - プロモーション動画で必ずこのシーンを強調
  - 「こんなにたくさんのキャラが動くの？」という驚きを演出

- **「放置して眺めるだけ」という手軽さ**
  - 「忙しい人でも楽しめる」「作業しながら眺められる」という訴求
  - 「5分設定して、あとは見守るだけ」という簡単さを強調

- **「かわいいメカ」というキャラクター性**
  - キャラクターデザインを前面に出す
  - 「このキャラかわいい！」という感情的なつながりを生む

## 3. 継続人口を増やす（リテンション戦略）

### 3.1 日次リテンション（毎日来てもらう）

**仕様**:
- **ログインボーナス**
  - 毎日ログインすると報酬（コイン、ユニット、装備など）
  - 連続ログイン日数に応じて報酬が増える（7日連続でレアユニットなど）
  - カレンダー形式で「あと○日で特別報酬」を視覚化

- **日替わりミッション**
  - 毎日3つの簡単なミッション（例：「1回バトルする」「10体撃破する」「ユニットを1体強化する」）
  - すべて達成すると特別報酬
  - 「今日のミッション」をホーム画面に大きく表示

- **日替わりイベント**
  - 毎日異なるイベント（例：「今日は遠距離型の経験値2倍」「今日はボス出現率アップ」）
  - 「今日は何の日？」という期待感を生む

### 3.2 週次リテンション（週に1回は来てもらう）

**仕様**:
- **週間チャレンジ**
  - 週替わりで3つのチャレンジ（例：「100体撃破」「5回バトル勝利」「新しいユニットを3体獲得」）
  - すべて達成するとレア報酬
  - 週の終わりに「今週の成果」を表示

- **週間ランキング**
  - 週ごとにリセットされるランキング
  - 上位者に特別報酬
  - 「今週は何位になれるか？」という競争心を刺激

### 3.3 長期的なリテンション（数ヶ月遊び続けてもらう）

**仕様**:
- **シーズンシステム（3ヶ月サイクル）**
  - 3ヶ月ごとにシーズンがリセット
  - シーズンごとに新しいユニットタイプ、装備、モードを追加
  - 「新しいシーズンが始まった！」という新鮮さを提供
  - シーズンランキングで特別報酬

- **コレクション要素の強化**
  - **図鑑システム**: 獲得したユニット、装備を記録
  - **コレクション達成度**: 「全ユニットの50%獲得」「全装備の30%獲得」など、段階的な目標
  - **「あと○体で全コレクション」という達成感**

- **実績システム（長期目標）**
  - 様々な条件を達成すると実績を獲得
  - 実績に応じた特別報酬、称号、バッジなど
  - 「100回バトル勝利」「1000体撃破」「全ユニットタイプ獲得」など、長期的な目標

- **ストーリー要素（将来的な拡張）**
  - 各ユニットタイプに簡単な背景ストーリー
  - 「このユニットの物語を完成させる」という目標
  - ストーリーを読むと報酬（コイン、経験値など）

### 3.4 コミュニティ形成（一緒に遊ぶ楽しさ）

**仕様**:
- **チーム共有システムの強化**
  - 自分の編成したチームを「おすすめ編成」として公開
  - 他のプレイヤーが使用すると、作成者に報酬
  - 「この編成すごい！」「真似してみたい」という発見と交流

- **ギルド・クランシステム（将来的な拡張）**
  - プレイヤーがグループを結成
  - ギルド内でランキング、報酬を共有
  - ギルド対戦イベント（週1回など）
  - 「仲間と一緒に」というコミュニティ感

- **チャット・コメント機能（将来的な拡張）**
  - ランキング画面で他のプレイヤーにコメント
  - 「おめでとう！」「すごい編成ですね」などの交流
  - ただし、モデレーション機能は必須（不適切なコメントを自動でフィルタ）

### 3.5 コンテンツの定期更新（飽きさせない）

**仕様**:
- **月1回の大型アップデート**
  - 新しいユニットタイプを追加（3-5体）
  - 新しい装備を追加（10-20個）
  - 新しいモードやイベントを追加

- **週1回の小型アップデート**
  - バランス調整
  - バグ修正
  - 新しいミッションやチャレンジ

- **「次回アップデート予告」**
  - アップデート前に「次はこんなユニットが追加されます！」と告知
  - 「楽しみ！」という期待感を生む

## 4. データドリブンな改善（継続的な最適化）

**仕様**:
- **離脱ポイントの分析**
  - どのタイミングで離脱するかデータを収集
  - 離脱率が高いポイントを特定し、改善

- **A/Bテストの実施**
  - オンボーディングの方法、報酬の量、UIの配置など、複数パターンをテスト
  - データに基づいて最適な方法を選択

- **プレイヤーフィードバックの収集**
  - ゲーム内で簡単にフィードバックを送れる機能
  - 「この機能が欲しい」「ここが分かりにくい」などの意見を収集
  - フィードバックに応じて改善

## 6. 成功指標（KPI）

### 6.1 チャレンジ人口の拡大
- **D1（1日目）リテンション率**: 目標50%以上
- **D7（7日目）リテンション率**: 目標30%以上
- **新規ユーザー獲得数**（月間）
- **友達招待数**
- **SNS共有数**

### 6.2 継続人口の増加
- **DAU**（日次アクティブユーザー数）
- **WAU**（週次アクティブユーザー数）
- **MAU**（月次アクティブユーザー数）
- **平均セッション時間**
- **平均プレイ日数**

### 6.3 エンゲージメント
- **1日あたりのバトル回数**
- **ログインボーナスの獲得率**
- **ミッション・チャレンジの達成率**
- **コレクション達成度**

## 7. 設計思想：長く遊び続けてもらうための原則

### 7.1 最初の5分で「ハマる」仕組み
- 即座にプレイ可能
- 視覚的なインパクト（200体の同時表示）
- 最初のバトルで必ず成功体験
- 最初の報酬で「また遊びたい」と思わせる

### 7.2 毎日来たくなる仕組み
- ログインボーナス（連続ログイン報酬）
- 日替わりミッション（簡単な目標）
- 日替わりイベント（「今日は何の日？」）

### 7.3 長期的な目標
- コレクション要素（全ユニット獲得など）
- 実績システム（様々な条件を達成）
- シーズンシステム（定期的な新鮮さ）

### 7.4 コミュニティ形成
- チーム共有（「この編成すごい！」）
- ランキング（競争心）
- 将来的にはギルド・クラン（一緒に遊ぶ楽しさ）

### 7.5 継続的なコンテンツ更新
- 月1回の大型アップデート
- 週1回の小型アップデート
