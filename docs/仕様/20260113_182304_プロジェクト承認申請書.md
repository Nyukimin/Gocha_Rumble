# プロジェクト承認申請書：Gocha Rumble 3D

**作成日**: 2026-01-13
**起案者**: プロデューサー
**対象**: 経営企画会議・承認委員会

---

## 1. エグゼクティブサマリー（要約）

本プロジェクト『Gocha Rumble 3D』は、**「操作しない贅沢」**と**「1000体の圧倒的映像体験」**を提供する、PC/スマホブラウザ向けの放置型シミュレーションゲームです。

*   **狙い**: 忙しい現代人に向けた「ながらプレイ」可能な癒やしと戦略のエンタメ。
*   **強み**: アプリインストール不要の手軽さと、SNS映えする「群集ビジュアル」。
*   **収益**: 広告モデル（常時表示＋動画リワード）により、サーバーコストを大幅に上回る高収益体質を実現。
*   **結論**: 初期投資を抑えつつ、長期間の収益が見込める高コスパ案件として、開発承認を申請します。

---

## 2. 製品コンセプト：なぜ今、このゲームか？

### 2.1 コンセプト：「わちゃわちゃ感」×「放置」
画面上を埋め尽くす1000体の「かわいいメカ」たちが、プレイヤーの指示通りに自動で戦う様子を眺めるゲームです。
*   **見る快感**: まるでアリの巣観察や、ドミノ倒しを見ているような、根源的な「見ていて気持ちいい」体験を提供します。
*   **手軽さ**: 面倒なアクション操作は一切なし。プレイヤーは「監督」として指示を出すだけです。

### 2.2 差別化要素
競合の「放置ゲーム」は2Dや地味な画面が多い中、本作は**最新のWeb 3D技術（Three.js）**を駆使し、ブラウザゲームとは思えないリッチな3D映像を実現します。
*   **社内評価**: ライト層からは「癒やされる」、ヘビー層からは「奥深い」と、両極端な層から高評価を得ています（社内ヒアリング済）。

## 3. ビジネスモデルと収益性

### 3.1 「勝手に黒字になる」収益構造
基本無料ですが、徹底した広告モデルで収益化します。

1.  **常時表示広告（ベース収益）**:
    *   PC画面の左右（余白）にバナーを常設。ゲームプレイを邪魔せず、遊んでいるだけでチャリンチャリンと収益が発生します。
2.  **リワード動画広告（ブースト収益）**:
    *   「ピンチからの復活」「ガチャ追加」など、ユーザーが**自ら喜んで見る**動画広告を導入。1再生あたりの単価が高く、収益の柱となります。

### 3.2 驚異的な利益率
Firebase（Googleのクラウド基盤）を活用し、運用コストを極限まで圧縮します。

*   **試算結果**: ユーザーが1回（10分）遊ぶと...
    *   **サーバー費用（支出）**: 約 **0.002円**
    *   **広告収入（収入）**: 約 **0.3円**
    *   **利益**: **+0.298円 （利益率 99%超）**

ユーザーが増えれば増えるほど、確実に利益が積み上がる構造です。赤字リスクは極めて低いです。

## 4. ターゲットと市場性

### 4.1 ターゲット層
*   **メイン**: 20〜40代のPC作業が多いオフィスワーカー。「仕事中にブラウザの別タブで開いておける」手軽さが刺さります。
*   **サブ**: スマホで手軽に暇つぶしをしたいライトゲーマー。

### 4.2 Webブラウザゲーム市場の再燃
近年、アプリストアの手数料（30%）を回避でき、インストール不要で即起動できる「Webゲーム」が再注目されています。
本作はURLをシェアするだけで友達と一緒に遊べるため、X（Twitter）やDiscordでのバイラル拡散（口コミ）が期待できます。

## 5. 技術的課題とリスク対策

承認者が懸念されるであろうリスクに対し、既に解決策を用意しています。

| リスク | 対策 |
| :--- | :--- |
| **「ブラウザで1000体も動くの？」** | **技術検証済**。`InstancedMesh`という描画技術と、簡易物理（嘘物理）を組み合わせ、低スペックPCでも快適に動くよう調整します。 |
| **「サーバー代が青天井では？」** | **予算アラートと設計で防止**。Firebaseの無料枠を最大限活用し、プレイ中のデータは端末側に保存する「ハイブリッド保存」で通信費をカットします。 |
| **「すぐ飽きられるのでは？」** | **AI学習システム**を導入。敵がプレイヤーの戦術を学習して対策してくるため、毎回違う展開になり、飽きさせません。 |
| **「広告が邪魔とクレームが来る？」** | **配置を配慮**。プレイ画面に被せないレイアウトを徹底し、動画広告は「見たい人だけが見る」方式にします（ヒアリングでも好評）。 |
