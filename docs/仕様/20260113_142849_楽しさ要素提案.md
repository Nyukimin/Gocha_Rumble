# 楽しさ要素提案 - ユーザー体験を向上させる追加機能

**作成日**: 2026-01-13  
**バージョン**: 1.0  
**ステータス**: 提案（検討中）  
**関連仕様**: `20260113_141959_製品仕様書.md`

## 1. 概要

本ドキュメントは、製品仕様書の検証結果に基づき、ユーザーが「楽しい」と感じる要素を追加提案するものである。

### 1.1 現状の強み（製品仕様書より）

1. **観戦の快感**: 「眺めて気持ちいい」を中心に据えた設計が一貫している
2. **意思決定の明確さ**: ロビー/章導入で決断、本編は放置という分離が明確
3. **ローグライト×ハクスラ**: 毎回違う体験と強化の快感を両立
4. **敵の適応システム**: プレイヤーの行動に反応する動的な体験

### 1.2 本提案の目的

- 現状の設計思想を壊さずに「楽しさ」を追加する
- 実装コストと効果のバランスを考慮した優先度を提示する
- コミュニティ形成・長期リテンションに寄与する要素を提案する

## 2. 提案一覧

### 2.1 視覚的カタルシスの強化

#### 2.1.1 「群れの一斉行動」イベント

- **ポイント**: 特定条件で味方全員が同時に攻撃/回避/進化する「シンクロモーメント」
- **意味**: 1000体が一斉に動く瞬間は、観戦ゲームとして最高の快感ポイント
- **例**:
  - 同時進化ラッシュ（条件を満たすと10体以上が連鎖進化）
  - 包囲殲滅（敵群を完全に囲んで一斉攻撃）
  - 緊急回避（ボス攻撃を全員が散開して避ける）
- **実装イメージ**:
  - 条件判定はサーバー/クライアントどちらでも可
  - 演出はカメラワーク＋スローモーション＋エフェクト強化
  - 発生頻度は1ランに1〜3回程度が適切

#### 2.1.2 「壊れビルド」の視覚化

- **ポイント**: シナジーが完成したとき、画面が派手に変わる演出
- **意味**: 「やった！壊れた！」という達成感を視覚で強調
- **例**:
  - 画面エフェクト変化（色調、パーティクル密度）
  - ダメージ数値のサイズ・色変化
  - BGMのレイヤー追加（壊れるほど音が重なる）
- **実装イメージ**:
  - シナジー完成フラグを検知してエフェクトレイヤーを追加
  - 段階的な演出（小シナジー→中シナジー→大シナジー）
  - パフォーマンスへの影響を考慮し、エフェクトはシンプルに

### 2.2 発見と収集の喜び

#### 2.2.1 図鑑システムの深化

- **ポイント**: 敵・同僚機・パーク・シナジーの図鑑を用意し、発見で報酬
- **意味**: 「次は何が見つかるか」という探索欲を刺激
- **例**:
  - 敵図鑑：適応パターンも記録（「この敵はこう学習する」）
  - シナジー図鑑：発見したシナジーの組み合わせを記録
  - 環境図鑑：各章の環境条件を記録
- **実装イメージ**:
  - Firestoreにユーザーごとの図鑑データを保存
  - 初回発見時にポップアップ＋報酬付与
  - 図鑑画面はロビーからアクセス可能

#### 2.2.2 隠しシナジー・レア組み合わせ

- **ポイント**: 通常では出ない「秘密の組み合わせ」を用意
- **意味**: コミュニティでの情報共有、発見の喜び
- **例**:
  - 3つ以上のパークが揃うと発動する「隠しシナジー」
  - 特定環境×特定パークで発動する「環境シナジー」
  - 発見時に図鑑に「???」から名前が明かされる演出
- **実装イメージ**:
  - シナジーテーブルに「隠し」フラグを追加
  - 発見までは図鑑に「???」で表示
  - コミュニティでの情報共有を促進（攻略サイト向け）

---

### 2.3 物語体験の強化

#### 2.3.1 テラリウムの環境ストーリーテリング

- **ポイント**: 章を進むごとに「何があったか」が断片的に分かる
- **意味**: 世界観への没入、「次の区画には何がある？」という期待
- **例**:
  - 各区画に残された「前任者の記録」（テキストログ）
  - 侵食体の「変異元」が何だったか示唆するビジュアル
  - 中枢に近づくほど「実験の真相」が見えてくる
- **実装イメージ**:
  - 章クリア時にテキストログを表示（スキップ可能）
  - 図鑑の「ストーリー」タブに蓄積
  - ビジュアルは背景オブジェクトで表現（低コスト）

#### 2.3.2 同僚機の背景

- **ポイント**: 捕獲した同僚機（元侵食体）に、元の姿・性格を示唆する設定
- **意味**: 仲間への愛着、「この子を活躍させたい」という感情
- **例**:
  - 同僚機ごとに短いフレーバーテキスト
  - 特定条件で「元の姿」を思い出す演出
  - 図鑑で同僚機の「発見ストーリー」を読める
- **実装イメージ**:
  - 同僚機マスターデータにフレーバーテキストを追加
  - 捕獲時に短いテキストを表示
  - 図鑑画面で詳細を閲覧可能

---

### 2.4 社会的楽しさ

#### 2.4.1 ハイライト自動生成・共有

- **ポイント**: ランの名場面を自動でハイライト化、SNS共有可能に
- **意味**: 「見て！」という共有欲求、コミュニティ形成
- **例**:
  - 壊れシナジー発動の瞬間
  - 大量同時撃破
  - ギリギリの撤退成功
  - 自動で10秒クリップを生成
- **実装イメージ**:
  - クライアント側でイベントログを記録
  - ラン終了時に「ハイライト候補」を抽出
  - WebGL録画またはリプレイデータからGIF/動画生成
  - Twitter/Discord共有ボタン

#### 2.4.2 ビルド共有・リプレイ

- **ポイント**: 成功したビルド構成を「レシピ」として共有
- **意味**: 「真似してみたい」「自分ならこうする」という遊び
- **例**:
  - ビルドコードを発行（文字列でコピペ可能）
  - 他人のビルドで「お試しラン」ができる
  - ランキング上位者のビルドを閲覧可能
- **実装イメージ**:
  - ビルド構成をBase64エンコードしたコードに変換
  - コード入力でビルドを再現
  - ランキング画面から「このビルドを見る」ボタン

### 2.5 チャレンジと達成感

#### 2.5.1 章クリア時の「作戦成功」演出

- **ポイント**: 章をクリアしたとき、明確な達成演出を入れる
- **意味**: 短い成功体験の積み重ねを強調
- **例**:
  - 「区画封鎖完了」の画面演出
  - 回収した同僚機の紹介カットイン
  - 次の章の「偵察情報」を予告
- **実装イメージ**:
  - 章クリア判定後に演出シーケンスを挿入
  - 2〜5秒程度の短い演出（スキップ可能）
  - 次章の環境条件をプレビュー表示

#### 2.5.2 縛りチャレンジ（オプション）

- **ポイント**: 自分でハンデを選んで挑む「縛りプレイ」報酬
- **意味**: 上級者向けのやりこみ、コミュニティでの自慢
- **例**:
  - 「同僚機なし」「特定パーク禁止」「撤退禁止」
  - 縛りを達成すると専用バッジ・称号
  - 縛りラン専用ランキング
- **実装イメージ**:
  - ロビーで「縛り条件」を選択可能
  - 縛り条件に応じてフラグを設定
  - クリア時に縛り達成を判定して報酬付与

### 2.6 サプライズ要素

#### 2.6.1 ランダムイベント（章間）

- **ポイント**: 章の導入時に低確率で発生する「特殊イベント」
- **意味**: 「今日は何が起きるか」というワクワク感
- **例**:
  - 「漂流物発見」（レアアイテム入手）
  - 「侵食体の内紛」（敵同士が戦っている状態でスタート）
  - 「前任者の遺産」（強力なパークを1つ獲得）
  - 「異常環境」（通常と違う環境条件）
- **実装イメージ**:
  - 章開始時に確率判定（5〜10%程度）
  - イベントテーブルからランダムに選択
  - イベント内容に応じて初期状態を変更

#### 2.6.2 隠し要素・イースターエッグ

- **ポイント**: 特定条件で発生する「隠しモード」「隠しエンディング」
- **意味**: 発見の喜び、コミュニティでの話題
- **例**:
  - 特定の同僚機編成で発生する「特殊シナリオ」
  - 全図鑑コンプリートで解放される「真の中枢」
  - 100回撤退で発生する「撤退マスター」称号
- **実装イメージ**:
  - 条件判定は実績システムと連動
  - 隠し要素は公式では告知しない（コミュニティ発見待ち）
  - 発見者には特別な称号を付与

---

### 2.7 カスタマイズの楽しさ

#### 2.7.1 自機の見た目カスタマイズ

- **ポイント**: 自機（オペレーター機）の外観を変更可能
- **意味**: 「自分だけの機体」という愛着
- **例**:
  - カラーバリエーション
  - パーツ変更（頭部、腕部など）
  - 達成報酬で解放されるレアスキン
- **実装イメージ**:
  - 自機モデルをパーツ分割して組み合わせ可能に
  - カラーはマテリアルパラメータで変更
  - スキンはFirestoreにユーザーデータとして保存

#### 2.7.2 テラリウム「ホーム」

- **ポイント**: ロビー画面をプレイヤーがカスタマイズできる
- **意味**: 「自分の空間」という居心地の良さ
- **例**:
  - 捕獲した同僚機がロビーで待機している
  - 達成した実績がトロフィーとして飾られる
  - 背景の装飾を変更可能
- **実装イメージ**:
  - ロビーシーンに配置オブジェクトを追加
  - 同僚機は最大5体程度を表示
  - 実績トロフィーは3Dオブジェクトとして配置

---

### 2.8 リプレイ性の強化

#### 2.8.1 「今日のシード」モード

- **ポイント**: 毎日変わる固定シードで全員が同じ条件でプレイ
- **意味**: 「今日の最高記録は誰？」という競争、毎日来る理由
- **例**:
  - 24時間限定のランキング
  - 同じシードなので攻略の比較が可能
  - 「今日のシード」専用報酬
- **実装イメージ**:
  - サーバー側で日付ベースのシードを生成
  - クライアントはシードを受け取って乱数を初期化
  - デイリーランキングをFirestoreに保存

#### 2.8.2 週替わり特殊ルール

- **ポイント**: 週ごとに異なる特殊ルールが適用されるモード
- **意味**: 「今週はどんなルール？」という新鮮さ
- **例**:
  - 「高速進化週間」（進化速度2倍）
  - 「捕獲祭り」（捕獲率アップ）
  - 「適応強化」（敵の学習が早い）
  - 「パーク大盤振る舞い」（パーク選択肢が5択に）
- **実装イメージ**:
  - 週ごとにルールIDをサーバーから配信
  - クライアントはルールIDに応じてパラメータを変更
  - 週間ランキングと連動

## 4. 設計思想との整合性チェック

各提案が製品仕様書の設計思想と矛盾しないことを確認する。

| 設計思想 | 整合性 | 備考 |
|----------|--------|------|
| 観戦・放置が主役 | OK | 全提案は観戦体験を強化する方向 |
| ゲーム中操作は最小限 | OK | 操作を増やす提案はなし |
| ロビー/章導入で意思決定 | OK | 新機能もロビーで選択する形式 |
| ラン内で強化、ラン間は幅 | OK | 図鑑・実績は「幅」に該当 |
| トレーディングは入れない | OK | 共有は「見せる」だけ、交換なし |
| 1000体運用 | 要注意 | 演出追加時はパフォーマンス検証必須 |
