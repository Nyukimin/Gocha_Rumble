# ユーザーのリクエストまとめ

## プロジェクト概要
- **タイトル**: Gocha Rumble 3D (GitHub: Gocha Rumble 3)
- **ジャンル**: 3Dオートバトラー / 放置ゲーム / コチャキャラ系
- **プラットフォーム**: Webブラウザ (PCでの閲覧を想定)

## コアコンセプト
1. **「ゴチャキャラ」感**: 画面上に大量のキャラクター（数千体規模）を表示。
2. **「かわいさ」**: かわいいメカデザイン（メカニカルな要素を持ちながらも可愛らしい見た目）。
3. **「見守る」楽しさ**: 最初の設定（ビルド）を行ったら、あとはゲームオーバーまで放置して眺めるだけ。
4. **視点操作**: 自由なカメラ操作が可能（360度、ズーム等）。

## ゲームシステム要件
- **ハクスラ要素**: ダメージ数値のポップアップ、ステータス強化。
- **ローグライク要素**: ランダムな戦利品やアップグレード要素。
- **プレイヤー**: 1体またはチームを編成してスタート。
- **ゲームループ**: セットアップフェーズ → バトルフェーズ（放置） → リザルト/ゲームオーバー。

## 技術・機能要件
1. **Google認証**: ユーザーログイン機能。
2. **非同期PvP（競争要素）**:
   - 自分のチームデータをサーバーに保存。
   - 他プレイヤーのデータ（ゴースト）と対戦できる機能。
   - ランキング/リーダーボードの実装。
3. **技術スタック**:
   - フロントエンド: Vite + Three.js (InstancedMesh使用で軽量化)
   - バックエンド: Firebase (Authentication, Firestore)

## デザイン・パフォーマンス要件

### ユニット表示数
- **目標**: 200体を同時表示
- **フレームレート**: 30FPS以上を目標

### ポリゴン数
- **一体あたり**: 300-400ポリゴンを目標
- **理由**: 200体表示時のパフォーマンス（30FPS維持）を考慮

### タイプ区別
- **区別方法**: 形状（シルエット、パーツ）で区別
- **色による区別**: 使用しない（チーム色を採用するため）

### 挙動・アニメーション
- **挙動スタイル**: SDガンダム的
  - 硬い、機械的な動き
  - 関節が明確に動く
  - 「ぐにょぐにょ」した柔らかい動きではない
